# Оптимизация сайта под Нейро/YandexGPT

## Контекст

Сайт ХИММЕТАЛЛИН (`C:\Users\mccaq\himmetallin-site`, Vite 6 + React 19 + TS) уже прошёл
базовую AEO/GEO-подготовку (см. `docs/superpowers/specs/2026-07-10-aeo-geo-design.md`):
JSON-LD `@graph` (Organization/WebSite/Service), `llms.txt`, полностью открытый
`robots.txt`. Эта работа была рассчитана на ИИ-ответники в целом (ChatGPT, Perplexity,
YandexGPT и т.п.).

Данная итерация — прицельная оптимизация под Нейро (ИИ-обзор в поиске Яндекса) и
YandexGPT. Ключевая находка: сайт — чистый client-side React SPA без пререндера/SSR
(`index.html` содержит только пустой `<div id="root">`, весь текстовый контент
рендерится через `main.tsx` после загрузки JS). Реальный видимый текст страницы
(заголовки, характеристики оборудования) недоступен краулерам, которые не выполняют
JS, — а это узкое место именно для того, чтобы Нейро могло прочитать и процитировать
контент, помимо уже существующего JSON-LD/`llms.txt`.

Сайт уже подтверждён в Яндекс.Вебмастере (`public/yandex_ab53b9442f3689f8.html`).
Прод — `https://chemmetalleng.com`, деплой по FTP на nic.ru через
`.github/workflows/deploy-nicru.yml` при пуше в `main`.

## Архитектура

### 1. Статический пререндер (Playwright-снапшот)

Новый файл `scripts/prerender.mjs`:

1. Поднимает локальный статик-сервер над уже собранным `dist/` (встроенный
   `http.createServer` Node, без новых зависимостей на раздачу статики).
2. Запускает headless Chromium через `playwright` (уже в `devDependencies` этого
   проекта — новая зависимость не нужна), открывает `http://localhost:<port>/`.
3. Ждёт состояния `networkidle`. Дополнительных задержек/имитации скролла не нужно:
   `Reveal` (`src/ui.tsx`) стартует с `shown=false` и переключается в `true` через
   `IntersectionObserver` сразу при монтировании для блоков, уже находящихся во
   вьюпорте, — то есть снапшот без скролла точно воспроизводит то состояние, которое
   позже соберёт клиентская гидрация. Рассинхрона (hydration mismatch) не будет.
4. Забирает `await page.content()` — полный `outerHTML` документа.
5. Перезаписывает `dist/index.html` этим HTML. Остальные файлы в `dist/`
   (JS/CSS/картинки) не трогаются — гидрация на клиенте продолжает работать по тем же
   путям, что и раньше.
6. Если Chromium не смог загрузить страницу, `networkidle` не наступил в разумный
   таймаут, или `page.content()` не удалось получить — скрипт завершается с
   ненулевым кодом (`process.exit(1)`), проваливая сборку. Тихий фолбэк на пустой
   `dist/index.html` недопустим: это должно быть заметно как красная сборка, а не
   молчаливая деградация.
7. В любом случае (успех/ошибка) корректно закрывает браузер и локальный сервер.

`package.json`: `"build"` меняется на
`"tsc --noEmit && vite build && node scripts/prerender.mjs"`.

`vite-plugins/seo-geo.ts` не меняется — он отрабатывает на хуках `vite build` до
`prerender.mjs` и уже вставляет JSON-LD/генерирует `llms.txt` в `dist/`; `prerender.mjs`
просто добавляет видимый текст поверх уже готового `dist/index.html` (JSON-LD в нём
сохраняется, т.к. Playwright открывает уже собранную страницу с этим тегом в `<head>`).

`src/main.tsx` и компоненты не меняются: `createRoot(...).render(...)` при реальной
загрузке в браузере пользователя гидрирует уже готовую разметку вместо создания её с
нуля. Визуальных и функциональных изменений для посетителей нет.

### 2. CI: `.github/workflows/deploy-nicru.yml`

Перед шагом `Build` добавляется шаг установки браузера:

```yaml
- name: Install Playwright browser
  run: npx playwright install --with-deps chromium
```

### 3. IndexNow (уведомление Яндекса о свежем контенте)

Один раз генерируется случайный ключ (hex-строка без дефисов, например через
`crypto.randomUUID().replace(/-/g, '')`) и кладётся в статический файл
`public/<ключ>.txt`, содержимое файла — сам ключ (требование протокола IndexNow).
После сборки файл автоматически попадает в `dist/` и заливается на прод по тому же
FTP, что и остальной сайт — на нём доступен по `https://chemmetalleng.com/<ключ>.txt`.

Новый шаг в `deploy-nicru.yml`, после шага `FTP deploy to nic.ru`:

```yaml
- name: Ping Yandex IndexNow
  if: env.HAS_FTP == 'true'
  run: |
    curl -sf "https://yandex.com/indexnow?url=https://chemmetalleng.com/&key=<ключ>&keyLocation=https://chemmetalleng.com/<ключ>.txt" || true
```

Решения:
- Бьём напрямую в `yandex.com/indexnow`, а не в общий шлюз `api.indexnow.org` — чтобы
  не зависеть от скорости репоста пинга общим шлюзом именно в сторону Яндекса, раз
  цель этой работы — прицельно Нейро/YandexGPT.
- `|| true` — намеренно: недоступность IndexNow не должна валить деплой, это
  ускоряющий сигнал, а не критичная часть публикации сайта.
- Пингуется один URL (`/`), т.к. сайт — одностраничник; сложный batch-JSON не нужен.
- Шаг выполняется только при `HAS_FTP == 'true'` — то есть только тогда, когда сайт
  реально был обновлён на проде (тот же гейт, что и у самого FTP-шага).

### 4. Проверка (`scripts/verify-seo-geo.mjs`)

Добавляется проверка, что после пререндера в `dist/index.html` есть реальный видимый
текст страницы, а не только пустой `<div id="root">`: например, что название
компании (`CONTACTS.company` из `src/content/services.ts`) или текст одного из
заголовков направлений встречается в HTML **вне** блока
`<script type="application/ld+json">`. Это ловит регресс пререндера прямо в CI, до
деплоя на прод.

## Данные и зависимости

Новых npm-зависимостей не требуется — `playwright` уже есть в
`devDependencies`. Новый ключ IndexNow — просто статический текстовый файл в
`public/`, не секрет (можно коммитить в репозиторий).

## Обработка ошибок

- Провал Playwright-снапшота → падение всей сборки (`npm run build`), а не тихий
  фолбэк на непререндеренный `dist/index.html`.
- Провал пинга IndexNow → игнорируется (`|| true`), деплой не должен зависеть от
  доступности стороннего API уведомлений.
- Провал новой проверки в `verify-seo-geo.mjs` → падение CI (тот же скрипт уже
  используется как gate, судя по существующим checks).

## Проверка (ручная, после деплоя)

1. `npm run build` локально — сборка проходит, `dist/index.html` содержит и JSON-LD,
   и видимый текст секций (не пустой `<div id="root">`).
2. `npm run preview` — визуально страница выглядит и ведёт себя как раньше (тикер,
   scroll-reveal анимации, форма).
3. Деплой в `main` → GitHub Actions: шаг установки Playwright проходит, сборка
   проходит, FTP-заливка проходит, шаг `Ping Yandex IndexNow` возвращает 200/202 в
   логах.
4. На проде: `curl https://chemmetalleng.com/` (без выполнения JS) — в сыром HTML
   виден реальный текст компании/секций. `curl https://chemmetalleng.com/<ключ>.txt`
   → 200, содержимое равно ключу.

## Вне рамок этой итерации

- Видимый блок «Частые вопросы» (FAQPage) — сознательно отложен пользователем (сильный
  сигнал для Нейро, но не в этой итерации).
- Общая Yandex SEO-выдача вне ИИ-обзора (Яндекс.Бизнес/Карты, Turbo-страницы,
  микроразметка отдельно от JSON-LD) — не входит в задачу, цель ограничена
  Нейро/YandexGPT.
- Свой чат-бот на YandexGPT API на сайте — не входит в задачу (это использование ИИ
  Яндекса как инструмента, а не оптимизация сайта под чужой ИИ).
- Настоящий SSR (`ReactDOMServer`, отдельный SSR-entry) — рассмотрен и отклонён в
  пользу Playwright-снапшота как менее инвазивного варианта для одностраничника.
